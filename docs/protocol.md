# Виды протокола:
1. ## Текстовый протокол:
	- ### Передача сообщений:
		-  Передача осуществляется через **TCP протокол**, который выступает *транспортным звеном* и передает *сырые байты* в протоколе.
		-  **Framing** сообщений определяется так:
			- Сначала отправляется **8 байтное беззнаковое**(`length_prefix`) число типа `64-bit unsigned integer` - длина впереди идущего сообщения. 
			-  После длины сообщения отправляется само *сообщение*(`payload`), **длина которого должна быть равна числу, отправленному ранее**.
			- То есть модель **framing** простая: `length_prefix + payload`. Стоит отметить, что `payload` это сразу **весь запрос целиком**.
 		 	> - При **передаче байтов по TCP** используем `network byte order/big-endian`. #TODO
	- ### Устройство запроса к серверу(`request`)
		- #### Описание и его содержание:
			- `request` представляем собой **клиентский запрос** на вызов удаленного метода.
			- `request` содержит **вид запроса(`request_type`), имя метода(`method_name`), версию протокола(`version`), поле `request_id`, поле `args_count`, передаваемые аргументы и опциональные метаданные**.
			- `request` можно разделить на 3 секции:
				- `start_line`: самая **первая строка запроса**. Содержит всего 3 значения: `request_type`, `method_name` и `version`. Каждое значение считается **отдельным словом** и отделяется **одним** пробелом от других.
				- `headers`: поля с метаданными запроса. Идут сразу после `start_line`. Метаданные должны быть описаны так: `name: value`. **секция headers заканчивается пустой строкой** `\r\n`
				- `args`: аргументы, которые передадутся в метод. **Идут сразу после завершения `headers`**, то есть после пустой строки. Их количество должно совпадать с значением в `args_count`, иначе вернется **ошибка** #TODO. Также аргументы должны передаваться со своими типами в виде: `type: arg`. Список со всеми типами [[types]] #TODO.
				- Каждая строка в запросе должна заканчиваться двумя подряд идущими символами `\r\n`. 
		- #### Пример валидного запроса:
			```text  
			CALL method_name RPC-TEXT/1.0  
			Request-Id: 42  
			Args-Count: 2  
				  
			int: 10  
			string: hello
			```
			
		- #### Обязательные поля запроса:
			- `request_type`: вид запроса. Тип: `string`.
			- `method_name`: имя метода/функции, который должен быть вызван. Тип: `string`.
			- `version`: версия протокола. Тип: `string`.
			- `request_id`: id текущего запроса. Тип: `32-bit unsigned integer`.
			- `args_count`: количество передаваемых аргументов. Тип: `8-bit unsigned integer`.
