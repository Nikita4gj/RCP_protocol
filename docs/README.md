# RPC протокол и его основные цели
---
## Для чего нужен:

- RPC **(Remote Procedure Call — удаленный вызов процедур)** - это ==технология и протокол взаимодействия между программами, позволяющий одной программе (клиенту) вызывать функции или процедуры на другом компьютере (сервере) так, будто они выполняются локально.==

 - **Применение:** Используется в *распределенных системах*, *микросервисной архитектуре* и для взаимодействия с *блокчейн-узлами*. 
---
## Принцип работы:

- Клиент вызывает **процедуру** -> параметры упаковываются (маршалинг) -> запрос *отправляется по сети -> сервер распаковывает и выполняет функцию -> результат отправляется обратно*.

- **Важно понимать**, что RPC это не протокол по передаче данных, а лишь **удаленный вызов некоторой функции**.  
---
## Цель написания этого протокола:

- В первую очередь этот протокол является *способом обучения* и моим личным **пет проектом**.

-  Понимание работы с сетью через такие низкоуровневые примитивы, как *системные сокеты и TCP протокол*.

- Получить опыт в проектировании *реальной рабочей системы*, которую можно будет где-то использовать.
